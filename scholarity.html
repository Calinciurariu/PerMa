1. Abstract

PerMa este o aplicatie de tipul magazin online de parfumuri. Aceasta aplicatie permite utilizatorului sa isi creeze un cont si sa cumpere dintr-o selectie de produse, acesta poate vizualiza produse. Totodata exista si un test pe care utilizatorul il poate folosi pentru a gasi un produs mai potrivit pentru acesta. Este si o parte de admin, acesta este un al-2-lea tip de utilizator care poate sa verifice stocul de produse curent.

2. Introduction

In momentul de fata, cumparatul online a devenit deja o normalitate in majoritatea cazurilor. Oricum, in ceea ce priveste industria de parfumuri, exista deja ceva timp de cand oamenii aleg sa cumpere parfumuri prin diverse platforme de vendorship, deoarece nu sunt atat de multe magazine care sa lucreze cu distribuitori mai interesanti sau exotici. Asadar, in final, o platforma pentru vanzarea si distribuirea de parfumuri se plaseaza intr-o nisa proprie si garanteaza interesul clientilor.

3. Structure

Structura platformei este bazata pe modelul bine cunoscut, MVC(Model-view-controller), ce este un model arhitectural. Succesul modelului se datoreaza izolarii logicii de bussines fata de considerentele interfetei cu utilizatorul, rezultand o aplicatie unde aspectul vizual sau/ si nivelele inferioare ale regulilor de bussines sunt mai usor de modificat, fara a afecta alte nivele. Pe langa MVC, elementul controlor(API) este accesat prin AJAX, prescurtarea pentru Asynchronous JavaScript and XML. Fiind o tehnica de programare pentru crearea de aplicatii web interactive rapide. O structurare in profunzime a aplicatiei sunt prezentate in sectiunile urmatoare.

Componentele Aplicatiei

  • 1. Front-End
    Front-End-ul se refere la toata partea de client ce cuprinde html/css/javascript unde gasim tot ce inseamna structura si aspectul paginilor web. Folosim AJAX pentru a modifica paginile fara a intrerupe interactiunea pentru utilizator. Site-ul avand ca scop prezentarea unor produse, datele sunt trimise si primite constant la server.

  • 2. Server
    Partea de server este scrisa in NodeJs folosind modulul http al limbajului. Avem un server REST simplu, care expune o serie de rute, acestea sunt grupate in functie de metoda de apelare. Avem o functie utilitara care construieste un raspuns dat bazat pe datele primite si pe datele extrase din baza de date. Utilizam modulul http, dar impachetat intr-un modul scris de noi, care este in sine o clona de express js. Modulul ne ofera o parte din functionalitatile cele mai importante ale framework-ului express, de la modalitatea simpla de a defini endpoint-uri pana la cateva utilitare care extrag path parameters si query parameters pentru noi, dar construiesc si body-ul, procesand octet stream-ul care vine by default.

  • 3. Data Handler
    Partea de management a bazei da date. Baza de date utilizata este de tipul mongoDB. Utilizam API-ul node js pentru executia instructiunilor pe o bazele de date de tipul mongoDB. Avem un handler generic deoarece mongodb ne permite sa lucram cu obiecte de orice forma, o flexibilitate aparte care este si principalul avantaj al MongoDb.
